## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/jquery-scrollbox.git`
3. Create a new branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes
6. Commit with clear messages: `git commit -m "Add: feature description"`
7. Push to your fork: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Development Setup

```bash
# Install dependencies
npm install

# Run linter
npm run lint

# Build the project
npm run build

# Run tests
npm test
```

## Code Style

- Use ES6+ syntax for modern JavaScript
- Follow the existing code style
- Run ESLint before committing: `npm run lint`
- Use meaningful variable and function names
- Add comments for complex logic

## Commit Messages

Use clear and descriptive commit messages:

- `Add: new feature description`
- `Fix: bug description (closes #123)`
- `Update: documentation changes`
- `Refactor: code improvement`
- `Test: add or update tests`

## Pull Request Guidelines

1. **Keep PRs focused** - One feature or fix per PR
2. **Update documentation** - If you change functionality, update README.md
3. **Test your changes** - Ensure everything works as expected
4. **Describe your changes** - Provide a clear description of what and why
5. **Reference issues** - If fixing a bug, reference the issue number

## Reporting Bugs

When reporting bugs, please include:

1. **Browser and version** - e.g., Chrome 120, Firefox 121
2. **Steps to reproduce** - Clear steps to reproduce the issue
3. **Expected behavior** - What should happen
4. **Actual behavior** - What actually happens
5. **Code sample** - Minimal code to reproduce the issue
6. **Screenshots** - If applicable

## Feature Requests

We welcome feature requests! Please:

1. Check if the feature already exists
2. Check if there's an open issue for it
3. Describe the use case clearly
4. Explain why it would be useful
5. Provide examples if possible

## Code of Conduct

- Be respectful and considerate
- Welcome newcomers and help them learn
- Focus on constructive criticism
- Accept feedback gracefully

## Questions?

- Open an issue with the `question` label
- Check existing issues and discussions
- Review the README.md for documentation
